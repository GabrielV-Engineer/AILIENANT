## AILIENANT क्या है?

**AILIENANT एक स्वायत्त कोडिंग एजेंट है जो VS Code के भीतर रहता है।** आप सहज भाषा में बताते हैं कि आपको क्या चाहिए; AILIENANT एक वास्तविक योजना लिखता है, बदलाव करता है, कोड को एक सैंडबॉक्स में चलाता है, परिणाम पढ़ता है और अपनी गलतियाँ खुद सुधारता है — और साथ ही अपनी सोच का हर कदम आपको दिखाता है।

लोकप्रिय AI सहायकों से यह **इस बात में अलग है कि यह कहाँ चलता है और कैसे निर्णय लेता है।** AILIENANT **local-first** है: यह पूरी तरह आपकी अपनी मशीन पर ओपन मॉडल (Ollama, LM Studio और अन्य) के साथ चल सकता है, और क्लाउड का सहारा केवल तभी लेता है जब किसी कार्य को वास्तव में इसकी आवश्यकता हो — और जब लेता है, तो डॉलर में बता देता है। आपके कोड को आपके लैपटॉप से बाहर जाने की ज़रूरत नहीं, और आप कभी किसी एक वेंडर से बंधे नहीं रहते।

> **एक पंक्ति में:** आपके कोडबेस के लिए एक निजी, लागत-सजग और पहले-योजना बनाने वाला AI इंजीनियर — ओपन सोर्स, बिना किसी वेंडर लॉक-इन के।

## लोग इसे क्यों इस्तेमाल करते हैं

- **🧠 कोड से पहले योजना।** एक समर्पित *प्लानर* आपके अनुरोध को एक ठोस स्पेसिफिकेशन और कार्य-सूची में बदलता है, दायरा (scope) तय कर देता है, और "ड्रिफ्ट" पर नज़र रखता है ताकि एजेंट चुपचाप भटककर आधा प्रोजेक्ट दोबारा न लिख दे। एक अलग *कोडर* उस योजना को अंजाम देता है। दो दिमाग, हर एक अपना काम बखूबी करता है।
- **🔒 आपका कोड आपका ही रहता है।** अपने मॉडल के साथ 100% लोकल चलता है। कोई अनिवार्य क्लाउड नहीं, कोई "घर फ़ोन करने वाली" टेलीमेट्री नहीं, आपके रिपॉज़िटरी पर कोई ट्रेनिंग नहीं।
- **💸 लागत दिखती है।** हर कार्य का एक लाइव टोकन-लेजर और एक कठोर बजट सीमा होती है। लोकल बनाम क्लाउड उपयोग और अनुमानित बचत छिपाई नहीं, दिखाई जाती है।
- **🪟 सोच दिखती है।** एक लाइव "थॉट बॉक्स" मॉडल की सोच को स्ट्रीम करता है, और एक चरण-दर-चरण ट्रेस हर पढ़ी गई फ़ाइल, चलाई गई कमांड और प्रस्तावित पैच को दिखाता है।
- **⏪ आप पीछे लौट सकते हैं।** किसी कार्य का हर चरण एक टिकाऊ चेकपॉइंट होता है। किसी विकल्प को आज़माने के लिए किसी भी बिंदु से शाखा (branch) बनाएँ — एजेंट के लिए सच्ची टाइम-ट्रैवल डिबगिंग।
- **🛡️ कोड सुरक्षित रूप से चलाता है।** उत्पन्न कमांड एक सैंडबॉक्स में चलते हैं (Docker, और WebAssembly व मानव-स्वीकृति वाले फ़ॉलबैक के साथ), कभी भी आपकी मशीन पर आँख मूँदकर नहीं।
- **🔌 कोई लॉक-इन नहीं।** अपना मॉडल और प्रदाता लाएँ — Ollama, LM Studio, vLLM, llama.cpp, OpenAI, Anthropic, Google, DeepSeek, Mistral और अधिक — और जब चाहें बदलें।

## यह कैसे अलग है?

| | **AILIENANT** | सामान्य क्लाउड सहायक |
| --- | --- | --- |
| पूरी तरह आपकी मशीन पर चलता है | ✅ Local-first, अपना मॉडल | ❌ केवल क्लाउड |
| पहले योजना, फिर कोड (दो-सिर) | ✅ प्लानर + कोडर, ड्रिफ्ट-गार्ड के साथ | ❌ एक मॉडल, एक प्रयास |
| स्मार्ट लोकल↔क्लाउड रूटिंग | ✅ काम कर सकने वाला सबसे सस्ता स्तर चुनता है | ❌ स्थिर |
| रियल-टाइम में लागत दिखाता है | ✅ टोकन-लेजर + बजट सीमा | ⚠️ अक्सर छिपी |
| टाइम-ट्रैवल / रन की शाखा | ✅ टिकाऊ चेकपॉइंट | ❌ स्टेटलेस |
| सैंडबॉक्स्ड निष्पादन | ✅ Docker / Wasm / स्वीकृति-आधारित | ⚠️ अक्सर होस्ट पर |
| वेंडर लॉक-इन | ✅ कोई नहीं — स्वतंत्र रूप से बदलें | ❌ एक से बँधा |

अधिक पूर्ण तकनीकी तुलना **[HowItWorks.md](HowItWorks.md)** में है।

## सुरक्षा और संरक्षण, डिज़ाइन से

AILIENANT यह मानकर चलता है कि कोई स्वायत्त एजेंट देर-सबेर ऐसा कुछ करने की कोशिश करेगा जो उसे नहीं करना चाहिए — और इसे रोकने के लिए ही इसे बनाया गया है।

- **डिफ़ॉल्ट रूप से पृथक।** कमांड एक पृथक Docker कंटेनर में चलती हैं (केवल-पढ़ने योग्य वर्कस्पेस, बिना नेटवर्क, बिना root), और Docker उपलब्ध न होने पर WebAssembly व मानव-हस्तक्षेप फ़ॉलबैक के साथ।
- **fail-closed अनुमतियाँ।** हर टूल को विशेषाधिकार के आधार पर वर्गीकृत किया जाता है; जो भी अपरिचित हो उसे **सुरक्षित साबित होने तक खतरनाक** माना जाता है, कभी उल्टा नहीं।
- **जहाँ ज़रूरी हो वहाँ मानव-स्वीकृति।** जोखिम भरे कार्य और बजट से अधिक खर्च आपकी स्पष्ट स्वीकृति के लिए रुक जाते हैं।
- **छेड़छाड़-प्रमाण ऑडिट लॉग।** स्वीकृतियाँ एक क्रिप्टोग्राफ़िक रूप से जुड़ी (blake2b) बही में दर्ज होती हैं जिसे आप सत्यापित कर सकते हैं।
- **मल्टी-टेनेंट पृथक्करण।** अनुक्रमित स्मृति का हर हिस्सा अपने वर्कस्पेस से नामबद्ध होता है, इसलिए प्रोजेक्ट कभी एक-दूसरे में नहीं रिसते।

## त्वरित शुरुआत

> पूर्ण मार्गदर्शिका: **[HowToUseIt.md](HowToUseIt.md)**

**पूर्व-आवश्यकताएँ:** Python 3.10+ (3.13 अनुशंसित), Node.js 20+, VS Code 1.85+, और कम से कम एक मॉडल स्रोत (एक लोकल Ollama/LM Studio इंस्टॉल, एक [LiteLLM](https://docs.litellm.ai/docs/simple_proxy) प्रॉक्सी, या क्लाउड API कुंजियाँ)।

```powershell
# 1. बैकएंड (ऑर्केस्ट्रेशन इंजन)
cd ailienant-core
python -m venv venv
.\venv\Scripts\activate          # Unix: source venv/bin/activate
pip install -r requirements.txt
copy ..\.env.example ..\.env     # Unix: cp ../.env.example ../.env

# 2. एक्सटेंशन (VS Code इंटरफ़ेस)
cd ..\ailienant-extension
npm install
npm run compile
```

फिर VS Code में प्रोजेक्ट खोलें और एक्सटेंशन शुरू करने के लिए **F5** दबाएँ। जब आप पहली बार AILIENANT सत्र खोलते हैं, तो यह आपके लिए बैकएंड शुरू कर देगा और आपके वर्कस्पेस को अनुक्रमित (index) करना शुरू कर देगा। अपने मॉडल अंतर्निहित **BYOM** पैनल से कॉन्फ़िगर करें, एक अनुरोध टाइप करें, और आप तैयार हैं।

## यह कैसे काम करता है (संक्षेप में)

```
आप पूछते हैं ─▶ प्लानर ─▶ ड्रिफ्ट ─▶ कोडर ─▶ सैंडबॉक्स इसे चलाता है
              (स्पेक +     गार्ड     (फ़ाइलें       ▲      │
               योजना)     (दायरा     संपादित)      │      ▼
                           लॉक)                 सुधारें ◀─ परिणाम पढ़ें
```

पर्दे के पीछे, एक स्टेटफुल **LangGraph** इंजन संदर्भ-व-जटिलता स्कोर के आधार पर हर कार्य को लोकल और क्लाउड मॉडल के बीच रूट करता है, **GraphRAG** (वेक्टर खोज + एक-छलांग निर्भरता ट्रैवर्सल) से सही फ़ाइलें लाता है, और हर चरण पर एक चेकपॉइंट सहेजता है ताकि कुछ भी न खोए। गहन संस्करण — आरेख, रूटिंग गणित, निष्पादन लूप और सुरक्षा मॉडल — **[HowItWorks.md](HowItWorks.md)** में है।
