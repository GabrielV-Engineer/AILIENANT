## AILIENANT 是什么？

**AILIENANT 是一个生活在 VS Code 里的自主编程智能体。** 你用自然语言描述需求；AILIENANT 会写出真正的计划、进行修改、在沙箱中运行代码、读取结果并自行纠错——而且会向你展示它推理的每一步。

它与主流 AI 助手的区别在于**它在哪里运行、如何决策。** AILIENANT 是**本地优先（local-first）**的：它可以完全在你自己的机器上、使用开放模型（Ollama、LM Studio 等）运行，只有当任务确实需要时才调用云端——并且会用美元金额告诉你何时调用了云。你的代码无需离开你的笔记本，你也永远不会被锁定在单一供应商。

你的代码值得拥有更多，而不仅仅是魔法般的自动补全或在黑盒中盲目运行的自主代理。Ailienant 是 VS Code 中唯一为追求严谨性、可审计性和完全掌控的工程师而打造的代理。

我们遵循 Spec-Driven Development（规格驱动开发）哲学。Ailienant 将你的架构读取为一个活图（GraphRAG），并运用最先进的推理能力——Dreaming、MCTS（蒙特卡洛树搜索）和 Harness Systems——跨多文件规划并执行复杂任务。这保证了精准无误，错误大幅减少，甚至克服了个人 PC 本地模型的上下文限制。

一切都以易用的方式集成，但不是"黑盒"。你制定规则：选择模型、更换执行引擎、控制预算、审计管道的每个步骤。Ailienant 自主执行，但在任何关键操作前（HITL）暂停等待你的批准。真正的自主性，零供应商锁定。你拥有自己的管道。

> **一句话：** 一个私有、成本可控、先规划的 AI 工程师，服务于你的代码库——开源且无供应商锁定。

## 人们为何使用它

- **🧠 先规划，后编码。** 一个由专职智能体组成的真正团队——*研究员（Researcher）*为你的代码绘制地图，*规划器（Planner）*把请求转化为具体的规格和任务清单并冻结范围，*编排器（Orchestrator）*驱动各个步骤，*编码器（Coder）*（以其 8 种专家角色之一）进行修改，还有一个可与之对话的*分析师（Analyst）*为你讲解代码库。漂移守卫确保智能体不会悄悄跑偏、重写半个项目。
- **🔒 你的代码仍归你。** 100% 本地运行，使用你自己的模型。没有强制云端、没有“回家”遥测、不在你的仓库上做训练。
- **💸 成本可见。** 每个任务都有实时的 token 账本和硬性预算上限。本地与云端用量以及预计节省都会显示出来，而非隐藏。
- **🪟 推理可见。** 实时的“思考框（Thought Box）”流式展示模型的推理，逐步轨迹显示每一次文件读取、命令执行和补丁提议。
- **⏪ 可以回退。** 任务的每一步都是持久化检查点。从任意点分叉以探索另一种方案——为智能体实现真正的时间旅行式调试。
- **🛡️ 安全地运行代码。** 生成的命令在沙箱中执行（Docker，并提供 WebAssembly 与人工审批回退），绝不盲目地对你的机器执行。
- **🔌 无锁定。** 自带模型与供应商——Ollama、LM Studio、vLLM、llama.cpp、OpenAI、Anthropic、Google、DeepSeek、Mistral 等——并可随时切换。

## 内部团队

AILIENANT 并非单一模型包揽一切——而是一个由专职成员组成的小团队，各司其职，由有状态的 **LangGraph** 引擎串联：

| 智能体 | 职责 |
| --- | --- |
| 🔭 **研究员** | 为你的代码库构建一张“骨架地图”——签名与跨模块关系——让规划器基于真实结构而非猜测来推理。 |
| 🧭 **规划器** | 将你的请求转化为具体且经校验的规格和任务清单（WBS），随后**冻结范围**，使工作不致蔓延。 |
| 🎛️ **编排器** | 逐步驱动计划，协调状态并将每一步路由到合适的模型层级。 |
| 🛠️ **编码器** | 进行实际修改——每个任务采用其 **8 种专家角色**之一。 |
| 💬 **分析师（Natt）** | 一个可与之对话的只读导师。它讲解你的代码以及 AILIENANT 本身，但从不触碰文件——是*声音*，而非*手*。 |

编码器会专化为每个任务所需的角色：**core-dev、架构/重构、devops/基础设施、secops、qa-测试、文档管理、版本控制、数据/ML 工程师**——各自拥有专属工具、护栏与审批触发器（例如，对 `.env` 的编辑始终会暂停等待你确认）。

当某一步失败时，**自愈**循环会读取错误并在放弃前提出修正补丁；对于开放式步骤，一个有界的 **ReAct 单元**会针对实时终端工作直至完成。完整的按智能体拆解见 **[HowItWorks.md](HowItWorks.md)**。

## 安全与防护，源于设计

AILIENANT 假定自主智能体迟早会尝试做它不该做的事——并据此构建以加以约束。

- **默认隔离。** 命令在隔离的 Docker 容器中执行（工作区只读、无网络、非 root），当 Docker 不可用时提供 WebAssembly 与人工介入回退。
- **fail-closed 权限。** 每个工具按权限分级；任何无法识别的东西都被视为**危险，直到被证明安全**，绝不反过来。
- **关键处需人工审批。** 高风险操作和预算超支会暂停，等待你的明确批准。
- **防篡改审计链。** 审批被记录在以密码学方式串联（blake2b）的账本中，可供你验证。
- **多租户隔离。** 每一段被索引的记忆都以其工作区命名，因此项目之间绝不互相泄漏。

## 快速开始

> 完整指南：**[HowToUseIt.md](HowToUseIt.md)**

**前置条件：** Python 3.10+（推荐 3.13）、Node.js 20+、VS Code 1.85+，以及至少一个模型来源（本地的 Ollama/LM Studio 安装、一个 [LiteLLM](https://docs.litellm.ai/docs/simple_proxy) 代理，或云端 API 密钥）。

```powershell
# 1. 后端（编排引擎）
cd ailienant-core
python -m venv venv
.\venv\Scripts\activate          # Unix: source venv/bin/activate
pip install -r requirements.txt
copy ..\.env.example ..\.env     # Unix: cp ../.env.example ../.env

# 2. 扩展（VS Code 界面）
cd ..\ailienant-extension
npm install
npm run compile
```

然后在 VS Code 中打开该项目并按 **F5** 启动扩展。首次打开 AILIENANT 会话时，它会**在一个自动分配的本地端口上为你启动后端**（一个空闲的 `127.0.0.1` 端口，例如 `http://127.0.0.1:59247/`）并自动把界面连接到它——无需配置端口。随后它会开始索引你的工作区。在内置的 **BYOM** 面板中配置你的模型，输入一条请求，即可开始。

> 想手动运行后端（无界面 / CI）？用 `uvicorn main:app --port 8000` 启动，并把扩展的 `backendUrl` 设置指向它。自动分配的端口仅用于 VS Code 内的常规流程。

## 工作原理（简版）

```
你提问 ─▶ 研究员 ─▶ 规划器 ─▶ 漂移守卫 ─▶ 编码器 ─▶ 沙箱运行
          (绘制       (规格      (范围       (编辑       ▲      │
           代码)       + 计划)    锁定)       文件)       │      ▼
                                                    自愈 ◀─ 读取结果
```

在幕后，有状态的 **LangGraph** 引擎根据上下文与复杂度评分，在本地与云端模型之间为每个任务进行路由——始终选择**能胜任的最便宜层级**，只有当任务确实需要时才调用云端。

它用 **GraphRAG** 取回正确的文件：它不会把整份文件塞进提示词，而是把你的代码索引为依赖图（Tree-sitter）并配以向量嵌入，然后通过向量检索 + 按重要性（PageRank）排序的 k 跳依赖遍历，仅取出相关的切片。这让提示词保持精简——**提示词体积平均缩减约 70%**——而这正是 AILIENANT 能在**普通硬件上良好运行**的原因：各层级的预算把上下文控制在小型本地模型的窗口内（最小可至 4K token），且索引常驻于快速的内存存储中。每一步都有检查点，确保不丢失任何内容。

**建立在规格之上，而非猜测之上。** 在触碰任何文件之前，规划者会将你的请求转化为一份冻结的 `MissionSpecification`——预期结果、范围、WBS 步骤、约束条件以及验收标准（包含 TDD 和 DDD 术语）。一旦冻结，规划者和编码者都无法悄然更改范围：`drift_monitor` 使用多因素相似度指标将每次重新规划与原始规格进行比对，如有偏差则升级由你决策。规格即合同；代理无法自行授权范围变更。

**失败被路由，而不会导致崩溃。** 每个代理轮次都运行在结构化执行挽具内：`reflexion_guard` 拦截异常并将其路由到专用的修复代理（而非显示堆栈追踪），确定性的 `finops_gate` 在图的每一步强制执行你的成本上限，结构化裁决——而非原始 stdout——驱动所有重试决策。若某个节点出现未处理的异常，会在错误传播前写入死信队列，让你能够检查并恢复。

深入版本——图示、完整的规格模式以及修复路由逻辑——见 **[HowItWorks.md](HowItWorks.md)**。

## 与你的代码对话：分析师

并非每个问题都需要智能体去*做*什么——有时你只是想理解。**分析师（Natt）**是一个驻留在侧边面板的对话伙伴：问它*“认证如何在这个服务中流转？”*、*“如果我改动这个函数会破坏什么？”*，甚至*“AILIENANT 的路由到底是怎么工作的？”*，它都会用通俗的语言回答。

它是一个**只读导师——是声音，绝非手。** 它解释、追踪、讲解，但从不编辑你的文件，因此你可以自由探索而不会被改动任何东西。

让它的回答可信的，是**它所依据的基础**——三个来源并用：你代码的**知识图谱**（以引用真实结构，而非幻觉）、你工作区的 **README**（以了解你项目的意图），以及 **AILIENANT 自身的产品文档**（以解释工具本身）。而且由于讲解比编码便宜，你可以从一个小巧的选择器中**挑选回答模型**——用快速的本地模型回答简单问题，用更强的模型做深入的架构讲解——而不影响检索质量。

## 看得见的记忆

AILIENANT 对你代码库的理解并非黑箱。内置**控制面板**将 GraphRAG 索引呈现为一张**交互式知识图谱**——一张力导向的文件及其依赖关系地图，其中连接最多的“枢纽”文件格外突出，相关模块共享一种颜色，重要性（PageRank）决定布局。配套的 2D **向量图**则投射出引擎在*语义*上如何对你的代码归类。这是一幅活生生的图景，展示智能体所知，以及它如何决定该读什么。

## 开放的生态

- **🧩 MCP 服务器。** AILIENANT 使用 **Model Context Protocol**，并附带一个经过甄选的可信服务器注册表（GitHub、Brave Search、Docker、Postgres），可一键启用。每个 MCP 工具都会**按权限分级**——未知者被视为危险，直到被证明安全——并且只有在你批准后才在该会话内被信任。
- **⚡ Skills。** 保存可复用的指令片段——全局或按工作区——并将其插入任意提示词。这是你自己的命令模板，随项目一同版本化。
- **🧰 工具。** 智能体通过一个带类型、按角色门控的工具注册表来行动：读取和追踪代码、以事务方式编辑文件、在沙箱中运行命令，并在不确定时询问你。该目录正**朝着约 56 个按角色分配的工具增长**（路线图见 **[docs/PROJECT_MANIFEST.md](docs/PROJECT_MANIFEST.md)**）；完整表格——哪个智能体使用哪个工具——见 **[HowItWorks.md](HowItWorks.md)**。

## Dreaming：在你离开时自我提升

编码是间歇性的——你出去吃午饭，你晚上下线。**Dreaming 模式**把这些空闲时间变成进展。你告诉 AILIENANT 去思考什么——*架构与模式*、*重构与技术债*、*缺陷修复*、整个工作区，或你自己输入的主题——在你离开期间，它会自主地围绕该焦点工作：研究代码、**把所学整合进长期记忆**，并探索改进。它会一路自我纠正，并在**错误开始累积时自行停止**。

至关重要的是，它**绝不会靠定时器自行唤醒来侵占你的机器**——是*你*在离开时启动它，从而决定何时花费资源。它**受预算上限约束**（一旦达到会话支出上限便会拒绝运行），并且安全：如果你回来并在某次处理途中保存了文件，该次处理会干净地中止而不写入。

选择适合你所休息时长的**配置档**——它们在速度、成本与深度之间取舍：

| 配置档 | 适合 | 大致 |
| --- | --- | --- |
| **Medium** | 午餐小憩——轻量、完全本地 | 1 个任务 · 3 个文件 · 约 60 分钟 |
| **Big** | 通宵——更深入、更多文件、本地 | 3 个任务 · 10 个文件 · 整夜 |
| **Cloud** | 顶级推理质量，受 token 限额约束 | 1 个任务 · 5 个文件 · token 封顶 |
| **Hybrid** | 云端*思考*、本地模型*编辑*——以更低成本换质量 | 2 个任务 · 6 个文件 |

完整机制——每个配置档究竟能达成什么、时间区间，以及离线树搜索（MCTS）如何校验候选改动——见 **[HowItWorks.md](HowItWorks.md)**。

## 实时终端与控制面板

智能体针对一个**持久、交互式的终端**工作——一个真实的 shell 会话，跨命令记住其工作目录与环境、实时流式输出并可被中断——全部都在沙箱内。**控制面板**（一个内置的、本地提供的仪表盘）为你提供对运行中会话的十一个视图：成本与路由遥测、硬件与运行时状态、记忆图谱、BYOM 模型、MCP 服务器与 skills、治理规则、用于审查待定补丁的暂存区、防篡改审计账本以及崩溃恢复。
