## Что такое AILIENANT?

**AILIENANT — это автономный агент-программист, который живёт внутри VS Code.** Вы описываете задачу обычным языком; AILIENANT составляет настоящий план, вносит изменения, запускает код в песочнице, читает результаты и исправляет собственные ошибки — показывая при этом каждый шаг своих рассуждений.

От популярных ИИ-ассистентов его отличает то, **где он работает и как принимает решения.** AILIENANT — **local-first**: он может полностью работать на вашей машине с открытыми моделями (Ollama, LM Studio и другие), обращаясь к облаку только тогда, когда задача действительно этого требует — и сообщает вам об этом в долларах. Вашему коду не нужно покидать ноутбук, и вы никогда не привязаны к одному поставщику.

> **Одной строкой:** приватный, экономный и сначала планирующий ИИ-инженер для вашей кодовой базы — с открытым исходным кодом и без привязки к вендору.

## Почему его выбирают

- **🧠 Сначала план, потом код.** Отдельный *Планировщик* превращает ваш запрос в конкретную спецификацию и список задач, фиксирует область работ и следит за «дрейфом», чтобы агент не ушёл в сторону и не переписал тихо половину проекта. Отдельный *Кодер* выполняет этот план. Две головы, каждая хорошо делает своё дело.
- **🔒 Ваш код остаётся вашим.** Работает на 100% локально с вашими моделями. Никакого обязательного облака, никакой телеметрии «домой», никакого обучения на вашем репозитории.
- **💸 Вы видите стоимость.** У каждой задачи есть живой учёт токенов и жёсткий лимит бюджета. Локальное и облачное использование и оценка экономии показываются, а не скрываются.
- **🪟 Вы видите рассуждения.** Живой «Ящик мыслей» транслирует рассуждения модели, а пошаговая трассировка показывает каждый прочитанный файл, выполненную команду и предложенный патч.
- **⏪ Можно отмотать назад.** Каждый шаг задачи — это устойчивая контрольная точка. Ответвитесь от любой точки, чтобы изучить альтернативу — настоящая отладка с путешествием во времени для агента.
- **🛡️ Безопасный запуск кода.** Сгенерированные команды выполняются в песочнице (Docker, с резервными вариантами WebAssembly и подтверждения человеком), а не вслепую на вашей машине.
- **🔌 Никакой привязки.** Используйте свою модель и поставщика — Ollama, LM Studio, vLLM, llama.cpp, OpenAI, Anthropic, Google, DeepSeek, Mistral и другие — и меняйте в любой момент.

## Безопасность и защита по умолчанию

AILIENANT исходит из того, что автономный агент рано или поздно попытается сделать то, чего не должен — и построен так, чтобы это сдержать.

- **Изоляция по умолчанию.** Команды выполняются в изолированном контейнере Docker (рабочая область только для чтения, без сети, не от root) с резервными вариантами WebAssembly и участием человека, когда Docker недоступен.
- **Права по принципу fail-closed.** Каждый инструмент классифицируется по уровню привилегий; всё нераспознанное считается **опасным, пока не доказано обратное**, а не наоборот.
- **Подтверждение человеком там, где это важно.** Рискованные действия и превышение бюджета приостанавливаются для вашего явного одобрения.
- **Защищённый от подделки журнал аудита.** Одобрения записываются в криптографически сцепленный (blake2b) журнал, который вы можете проверить.
- **Многоарендная изоляция.** Каждый фрагмент проиндексированной памяти привязан к своей рабочей области, поэтому проекты никогда не «протекают» друг в друга.

## Быстрый старт

> Полное руководство: **[HowToUseIt.md](HowToUseIt.md)**

**Требования:** Python 3.10+ (рекомендуется 3.13), Node.js 20+, VS Code 1.85+ и хотя бы один источник моделей (локальная установка Ollama/LM Studio, прокси [LiteLLM](https://docs.litellm.ai/docs/simple_proxy) или облачные API-ключи).

```powershell
# 1. Бэкенд (движок оркестрации)
cd ailienant-core
python -m venv venv
.\venv\Scripts\activate          # Unix: source venv/bin/activate
pip install -r requirements.txt
copy ..\.env.example ..\.env     # Unix: cp ../.env.example ../.env

# 2. Расширение (интерфейс VS Code)
cd ..\ailienant-extension
npm install
npm run compile
```

Затем откройте проект в VS Code и нажмите **F5**, чтобы запустить расширение. При первом открытии сессии AILIENANT он сам запустит бэкенд и начнёт индексировать вашу рабочую область. Настройте модели в встроенной панели **BYOM**, введите запрос — и можно работать.

## Как это работает (коротко)

```
Вы спросили ─▶ Планировщик ─▶ страж ─▶ Кодер ─▶ песочница выполняет
              (пишет спец.    дрейфа    (правит       ▲      │
               + план)        (область   файлы)       │      ▼
                               зафикс.)           исправь ◀─ читает результат
```

Под капотом движок **LangGraph** с состоянием маршрутизирует каждую задачу между локальными и облачными моделями по оценке контекста и сложности, извлекает нужные файлы через **GraphRAG** (векторный поиск + обход зависимостей на один шаг) и сохраняет контрольную точку на каждом шаге, чтобы ничего не терялось. Подробная версия — диаграммы, математика маршрутизации, цикл выполнения и модель безопасности — в **[HowItWorks.md](HowItWorks.md)**.
